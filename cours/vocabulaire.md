# Définitions / Vocabulaire

## Client/Serveur/protocole réseau

- Client : application qui envoie une requête à un serveur
- Serveur : application qui reçoit une requête d'un client et lui répond
- Protocole réseau : ensemble de règles qui régissent les échanges entre un client et un serveur (ex : HTTP, FTP, SMTP...)
## Front-end/Back-end/Back office
- Front-end : partie visible et interactive d'une application web, côté client (navigateur)
- Back-end : partie serveur d'une application web, qui gère la logique métier, les données et la communication avec le front-end
- Back office : interface d'administration d'une application web, accessible uniquement aux utilisateurs autorisés (ex : tableau de bord, gestion des contenus)
## Navigateur/cookies/session
- Navigateur : logiciel qui permet d'afficher et d'interagir avec des pages web (ex : Chrome, Firefox, Safari...)
- Cookies : petits fichiers stockés par le navigateur sur l'ordinateur de l'utilisateur, qui permettent de conserver des informations entre les sessions (ex : préférences, identifiants...)
- Session : période de temps pendant laquelle un utilisateur interagit avec une application web, généralement identifiée par un identifiant unique stocké dans un cookie

## UI/UX/VUI

- UI (User Interface) : ensemble des éléments visuels et interactifs d'une application web, qui permettent à l'utilisateur de naviguer et d'accomplir des tâches
- UX (User Experience) : ensemble des perceptions et des émotions ressenties par un utilisateur lors de son interaction avec une application web, qui influencent sa satisfaction et sa fidélité
- VUI (Voice User Interface) : interface utilisateur basée sur la reconnaissance vocale, qui permet à l'utilisateur d'interagir avec une application web en utilisant la voix (ex : assistants vocaux comme Alexa, Google Assistant...)

## Mobile-First

- Approche de conception et de développement d'une application web qui privilégie d'abord l'expérience utilisateur sur les appareils mobiles (smartphones, tablettes) avant de l'adapter aux écrans plus grands (ordinateurs de bureau)

## Application Web progressives (PWA)

- Applications web qui utilisent des technologies modernes pour offrir une expérience utilisateur similaire à celle des applications natives, avec des fonctionnalités telles que le mode hors ligne, les notifications push et l'installation sur l'écran d'accueil

## Pages mobiles accélérées (AMP)

- Framework open-source développé par Google pour créer des pages web mobiles rapides et optimisées, en utilisant un ensemble de règles et de composants spécifiques

## "Low-code" et "no-code" / CMS

- Low-code : approche de développement d'applications web qui utilise des outils visuels et des composants préconstruits pour réduire la quantité de code à écrire, permettant ainsi aux développeurs de créer des applications plus rapidement
- No-code : approche de développement d'applications web qui permet aux utilisateurs non techniques de créer des applications en utilisant des interfaces visuelles et des outils de glisser-déposer, sans avoir besoin de coder
- CMS (Content Management System) : système de gestion de contenu qui permet de créer, gérer et publier du contenu sur un site web sans avoir besoin de connaissances en programmation (ex : WordPress, Joomla, Drupal)
